# 步步原型设计

这是一个多端教学产品原型仓库，当前包含教师端、学生端和白板端三部分。仓库中的图片是各端页面的设计参考，教师端已经基于图片完成了第一版可运行原型，后续开发应继续以这些图片为视觉和交互基准。

## 目录结构

```text
.
├── 教师端/
│   ├── teacher-app/          # 已实现的教师端 React/Vite 原型
│   └── *.png                 # 教师端页面设计参考图
├── 学生端/
│   └── S01-S16_*.png         # 学生端页面设计参考图
└── 白板端/
    ├── B01-B14_*.png         # 白板端页面设计参考图
    ├── _contact_sheet_B01_B14.jpg
    └── 白板端拆页说明.md
```

## 当前状态

- 学生端：已有 16 张页面参考图，待开发。

## 启动教师端

进入教师端应用目录：

```powershell
cd 教师端\teacher-app
npm install
npm run dev
```

默认访问地址通常是：

```text
http://127.0.0.1:5173
```

如果端口被占用，Vite 会自动切换到下一个端口，以终端输出为准。

## 后续开发建议

### 分支策略

建议按端拆分功能分支：

```text
main                    # 稳定基线
feature/student-app     # 学生端开发
feature/whiteboard-app  # 白板端开发
feature/shared-ui       # 跨端通用组件或样式
```

如果需要并行开发，优先使用 Codex 原生 worktree 或 Git worktree，让不同端的开发互相隔离。每个 worktree 对应一个分支，每个 Codex 线程只负责一个端。

### 开发顺序

1. 先参考教师端现有实现，复用项目技术栈和视觉语言。
2. 学生端按 `S01` 到 `S16` 的页面顺序开发。
3. 白板端按 `B01` 到 `B14` 的页面顺序开发，并结合 `白板端拆页说明.md`。
4. 如果发现多个端有重复的导航、按钮、弹窗、面板、卡片等，再抽取共享组件。
5. 每完成一组页面，运行本地服务并做视觉核对。

### 视觉还原要求

- 设计图片是主要依据，页面布局、信息层级、颜色、间距和交互状态应尽量贴近参考图。
- 教师端已完成页面可作为实现风格参考，但不要为了复用而牺牲学生端和白板端各自的场景特征。
- 新增页面应优先保证首屏完整、文字不溢出、按钮和面板状态清晰。
- 白板端属于大屏授课场景，应优先考虑横屏、投屏和触控操作的可读性。
- 学生端属于平板/学习终端场景，应优先考虑任务路径清楚、输入和提交动作明确。

## 技术约定

教师端当前技术栈：

- React 18
- Vite
- React Router
- Tailwind CSS
- lucide-react

后续学生端和白板端如果没有特殊原因，建议沿用相同技术栈，减少维护成本。

建议每个端保持独立应用目录，例如：

```text
教师端/teacher-app
学生端/student-app
白板端/whiteboard-app
```

这样可以独立启动、独立构建，也便于未来按端部署。

## 提交流程

开发前先确认当前状态：

```powershell
git status
```

完成一组明确改动后提交：

```powershell
git add .
git commit -m "Implement student home prototype"
git push
```

提交信息建议用英文短句，说明实际完成的页面或能力。

## 不应提交的内容

以下内容已经通过 `.gitignore` 排除：

- `node_modules/`
- `dist/`
- `.vite/`
- `.env` 和 `.env.*`
- 日志文件
- 临时截图和调试脚本

如果后续新增生成产物或本地临时文件，应同步加入 `.gitignore`。
